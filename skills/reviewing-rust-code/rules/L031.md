# Rule: loop-without-exit-bound

**Severity:** warning

Inspired by [Power of 10 Rule 2](https://en.wikipedia.org/wiki/The_Power_of_10:_Rules_for_Developing_Safety-Critical_Code): *"Give all loops a fixed upper bound."*

## Problem

A `loop` or `while` construct has no clear upper bound on iterations. If the exit condition depends on external state (network, user input, hardware) or a subtle invariant, the loop may spin indefinitely — hanging the process or consuming 100% CPU.

## Example

### Bad
```rust
loop {
    let status = check_service(&client).await;
    if status.is_ready() {
        break;
    }
    tokio::time::sleep(Duration::from_secs(1)).await;
    // If the service never becomes ready, this runs forever
}
```

### Good
```rust
let deadline = Instant::now() + Duration::from_secs(60);
loop {
    let status = check_service(&client).await;
    if status.is_ready() {
        break;
    }
    if Instant::now() >= deadline {
        return Err(Error::Timeout);
    }
    tokio::time::sleep(Duration::from_secs(1)).await;
}
```

Or with an iteration cap:
```rust
for attempt in 0..MAX_RETRIES {
    let status = check_service(&client).await;
    if status.is_ready() {
        return Ok(());
    }
    tokio::time::sleep(Duration::from_secs(1)).await;
}
Err(Error::RetriesExhausted)
```

## When to flag

- `loop { ... }` where the `break` condition depends on external state without a timeout or iteration cap.
- `while` condition that depends on a value modified inside the loop body, with no guaranteed convergence and no upper bound.
- Retry loops without a maximum attempt count or deadline.
- Polling loops without a timeout.

## When NOT to flag

- `for x in iter` — iterator-driven loops are bounded by the iterator.
- `loop` in an event loop or server accept loop that is *designed* to run indefinitely (e.g., `tokio::main`, `loop { conn = listener.accept().await }`).
- `while let Some(item) = stream.next().await` consuming a bounded stream/channel.
- Loop with a clear termination proof from the surrounding logic (e.g., each iteration removes an element from a shrinking collection).
