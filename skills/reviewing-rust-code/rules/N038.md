# Rule: name-scope-mismatch

**Severity:** warning

Inspired by [Linux kernel coding style §4](https://www.kernel.org/doc/html/v6.4/process/coding-style.html#naming): *"C is a Spartan language, and so should your naming be... LOCAL variable names should be short, and to the point. GLOBAL functions need descriptive names."*

## Problem

A public function, type, or constant has a short, cryptic name that forces callers to read the implementation to understand the contract. Or conversely, a local variable in a short scope has an unnecessarily verbose name that clutters the code. The mismatch between name length and scope size reduces readability in both directions.

## Example

### Bad
```rust
// Public function with opaque name — callers cannot understand
// the contract without reading the implementation.
pub fn calc(o: &Order, a: &Address) -> Result<Money> {
    // ...
}

// Local variable in a 3-line loop with an unnecessarily verbose name
for current_item_index in 0..items.len() {
    process(&items[current_item_index]);
}
```

### Good
```rust
pub fn calculate_shipping_cost(order: &Order, destination: &Address) -> Result<Money> {
    // ...
}

for i in 0..items.len() {
    process(&items[i]);
}
```

## When to flag

- Public function, type, trait, or constant with a name that is an abbreviation or acronym not widely understood outside the project (e.g., `proc`, `calc`, `mgr`, `ctx` as public names).
- Public API where parameter names are single letters or meaningless abbreviations (e.g., `pub fn send(a: &str, b: &str, c: bool)`).
- The name requires reading the function body or doc comment to understand what it does.

## When NOT to flag

- Short local variable names in short scopes (`i`, `n`, `buf`, `tx`, `rx` in a few-line block).
- Domain abbreviations that are standard within the project or industry (e.g., `tcp`, `http`, `db`, `io`).
- Conventional names like `Self`, `self`, `new()`, `default()`.
- Iterator variables and closure parameters where the type provides sufficient context (e.g., `items.iter().map(|x| x.value)`).
- Private/internal functions where the call site is close and the module name provides context.
