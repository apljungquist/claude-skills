# A008: silent-saturation

**Level:** default

## Problem

Numeric cast via `as` silently truncates or saturates the value.
In Rust, `as` casts between integer types are defined to truncate (wrapping)
for same-or-smaller-width and saturate for float-to-int,
which can silently produce incorrect values.
The program silently uses a truncated or wrapped value —
producing wrong sizes, negative-as-huge-positive indices,
or incorrect arithmetic downstream.

## Pros / Cons / Decision

**Pros of using `try_from` instead of `as`:**
- Invalid conversions produce an error instead of silent data corruption.
- Code is explicit about which conversions are expected to succeed.

**Cons:**
- `try_from` returns `Result`, adding error handling overhead.
- Widening casts (`u8` → `u32`) are always safe and don't need `try_from`.

**Decision:**
Use `try_from` for narrowing or sign-changing casts.
`as` is acceptable for widening casts and intentional truncation with a comment.

## Example

### Bad
```rust
let file_size: u64 = get_file_size();
let buf_size: usize = file_size as usize; // Truncates on 32-bit platforms

let count: i64 = -1;
let index: usize = count as usize; // Wraps to usize::MAX
```

### Good
```rust
let file_size: u64 = get_file_size();
let buf_size: usize = usize::try_from(file_size)
    .context("file too large for this platform")?;

let count: i64 = -1;
let index: usize = usize::try_from(count)
    .context("count must be non-negative")?;
```

## When to flag

- `as usize` from `u64`/`i64`/`u128`/`i128` — truncates on 32-bit targets.
- `as u8`/`as u16`/`as i8`/`as i16` from larger integer types without prior range check.
- `as` cast from signed to unsigned (or vice versa) where negative values are possible.
- `as usize` from `f64`/`f32` (saturates to 0 or MAX).

## When NOT to flag

- Widening casts (e.g., `u8 as u32`, `i16 as i64`) — these are lossless.
- Cast is preceded by a bounds check or clamp that guarantees the value fits.
- The code only targets 64-bit platforms and documents this assumption.
- `as` cast in bitwise/low-level code where truncation is intentional and documented.
